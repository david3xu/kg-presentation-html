Path-Based Query - Power of Traversal

Finding connections between entities:

SPARQL (variable path length):

SELECT ?person1 ?person2 ?path
WHERE {
  ?person1 kg:name "Albert Einstein" .
  ?person2 kg:name "Stephen Hawking" .
  ?person1 (kg:influenced)+ ?person2 .
}

Cypher (showing the actual path):

MATCH path = (p1:Person {name: "Albert Einstein"})-[:INFLUENCED*]->(p2:Person {name: "Stephen Hawking"})
RETURN [node in nodes(path) | node.name] as connectionPath

Path queries reveal connections that would be extremely difficult to express in traditional query languages, especially with variable-length paths.
